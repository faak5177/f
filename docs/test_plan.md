# План тестирования

Автотесты: `tests/test_scenarios.py`. Запуск:

```bash
pytest -q
```

## Гость

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| TG-1 | Открыть `/login/` | 200, в HTML есть «Вход в систему» |
| TG-2 | Войти с правильными admin-данными | 302 → `/products/` |
| TG-3 | Войти с неверными данными | 200, остаёмся на странице входа |
| TG-4 | Открыть `/` | 200/302 |
| TG-5 | Открыть `/orders/` | 302 → `/login/` |
| TG-6 | Открыть `/products/` | 302 → `/login/` |

## Клиент (role_id = 3)

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| TC-7 | Открыть `/products/` | 200, виден каталог |
| TC-8 | Открыть `/products/create/` | 302 (нет прав) |
| TC-9 | Открыть `/orders/` | 200, видны только свои заказы |

## Менеджер (role_id = 2)

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| TM-10 | Панель фильтров на `/products/` | В HTML есть `class="filters"` и `name="q"` |
| TM-11 | Поиск по строке | 200 |
| TM-12 | Фильтр по производителю | 200 |
| TM-13 | Сортировка по цене (asc) | 200 |
| TM-14 | Открыть `/orders/` | 200 |
| TM-15 | Открыть форму `/products/create/` | 200 |

## Администратор (role_id = 1)

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| TA-16 | Кнопка «Добавить товар» | Видна на `/products/` |
| TA-17 | Открыть `/products/create/` | 200, в HTML «Новый товар» |
| TA-18 | Создать товар, затем удалить | 302 на create, 302 на delete, в БД нет |
| TA-19 | Открыть `/orders/` | 200, видны все заказы |
| TA-20 | Редактировать количество в заказе | quantity в БД обновилось |
| TA-21 | Создать + удалить заказ | count +1, затем -1 |
| TA-22 | Выход `/logout/` | 302 → `/login/` |

## Негативные проверки

- Гость не имеет доступа ни к `/products/`, ни к `/orders/` (TG-5, TG-6).
- Клиент не может вызвать `/products/create/` (TC-8).
- Авторизация с неверными данными не выдаёт редирект (TG-3).
