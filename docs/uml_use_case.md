# UML Use Case — диаграмма прецедентов

```mermaid
flowchart LR
    Guest((Гость))
    Client((Клиент))
    Manager((Менеджер))
    Admin((Администратор))
    System((Система))

    UC1[Авторизоваться]
    UC2[Просматривать каталог]
    UC3[Искать / фильтровать / сортировать товары]
    UC4[Оформить заказ]
    UC5[Просматривать свои заказы]
    UC6[Редактировать заказ]
    UC7[Просматривать все заказы]
    UC8[Добавлять / редактировать товары]
    UC9[Удалять товары / заказы]
    UC10[Выявлять товары с остатком = 0]
    UC11[Подсвечивать скидки > 12%]

    Guest --> UC1
    Client --> UC1
    Client --> UC2
    Client --> UC3
    Client --> UC4
    Client --> UC5

    Manager --> UC1
    Manager --> UC2
    Manager --> UC3
    Manager --> UC6
    Manager --> UC7
    Manager --> UC8

    Admin --> UC1
    Admin --> UC2
    Admin --> UC3
    Admin --> UC6
    Admin --> UC7
    Admin --> UC8
    Admin --> UC9

    System --> UC10
    System --> UC11
```

## Описание ролей

| Роль | role_id | Доступные действия |
| --- | --- | --- |
| Гость | — | Только форма входа |
| Клиент | 3 | Каталог, поиск, оформление и просмотр своих заказов |
| Менеджер | 2 | Всё выше + редактирование товаров и заказов, просмотр всех заказов |
| Администратор | 1 | Всё выше + удаление товаров и заказов |
